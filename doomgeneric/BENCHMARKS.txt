# Doom Hardware Accelerator: Expected Gains

## Current Baseline (Software Rendering)
- **Resolution:** 640 x 400
- **Platform:** Zynq UltraScale+ ZU3EG (Cortex-A53)
- **Render Time:** ~14.3 ms per frame
- **Max FPS:** ~70 FPS
- **Bottleneck:** CPU Pixel Filling (R_DrawColumn / R_DrawSpan)

## Expected Hardware Performance
Moving the rendering logic (`R_DrawColumn` / `R_DrawSpan`) to the FPGA (Programmable Logic) typically yields:

### 1. 640 x 400 Resolution
- **Render Time:** < 1 ms
- **Total Frame Time:** ~3-4 ms (Game Logic + Overhead)
- **Max FPS:** ~250 - 300 FPS
- **Gain:** **3.5x - 4x Speedup**

### 2. High Resolution (1280x720 or 1920x1080)
This is where Hardware Acceleration truly shines. Software rendering scales linearly with pixel count (more pixels = much slower), while FPGA pipelines can handle pixels in parallel or at wire speed.

| Resolution | Software FPS (Est.) | Hardware FPS (Est.) | Gain Factor |
|------------|---------------------|---------------------|-------------|
| 640x400    | 70 FPS              | 250+ FPS            | 3.5x        |
| 1280x720   | 25 FPS              | 100+ FPS            | 4x          |
| 1920x1080  | 10 FPS              | 60+ FPS             | **6x**      |

## Development Strategy
1.  **Profile:** Use the current TCP metrics to benchmark game logic vs. rendering.
2.  **Direct Video:** Replace TCP with writing to a physical memory buffer (VDMA) for HDMI output.
3.  **HLS/Verilog:** Implement `R_DrawColumn` in hardware to write to that buffer directly.
